1)  프론트 그리는 순서

  1. material - 안드로이드/cupertino - ios 중에서 하나를 선택
  2. Scaffold로 감싼다
  3. 그리고 이제 제작한다.

2) 생성자의 매개변수에 따라 달라지는 화면

Java에서 배웠던 것처럼 dart에서도 생성자 오버로딩의 개념으로 각각의 객체들의 속성값을 다르게 가져갈 수 있다. 화면에서는 Container 객체를 하나 생성한 다음에 거기에 color: 속성에다가 빨간색의 속성을 집어 넣어서 화면을 붉게 만들었다.

MaterialApp 객체의 여러가지 속성들을 확인할 수 있다. 


3) assets 디렉터리 생성 후, 그림 파일들 경로 설정


pubspec.yaml 파일에 들어가서 경로 설정을 해주어야 한다.


원래 asssets 적혀 있는 각각의 경로를 지우고, assets 디렉터리 내의 모든 파일을 공유하기 위해서 설정을 바꿔 주어야 한다.


저렇게 바꿔 준 상태에서 오른쪽 상단의 Pub get을 눌러주면 적용이 완료된다.



4) Column과 Row

Column과 Row는 컴포넌트나 위젯 배치에서 각각 세로와 가로로 화면에 출력되게 만든다.







5. 여러가지 위젯


ctrl + enter 단축키


공간값 주기 : Spacer(), Row()의 mainAxisAlignment 속성



Spacer()는 공간을 잡아먹는 객체이다. UI 화면에서 남은 공간이 있으면 그 공간은 Spacer()를 이용하면 공간을 차지하게 된다.




padding 값 주기

alt + enter를 이용하면 현재 객체를 Padding()으로 감싸면서 padding 값을 줄 수 있다.
